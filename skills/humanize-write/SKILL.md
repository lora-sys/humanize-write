# Humanize Write

把AI草稿变成「像人写的」成品的优化流程。

## 快速开始

```
写手窗口：给我写一篇关于[主题]的[格式]，受众是[受众]，目的是[目标]。
参考材料：[粘贴你的笔记/大纲/引语]

规则：见 RULES.md
```

## 完整工作流

**V0 → 裁判审查 → V1 → 人工润色V2**

### 第一步：写手生成V0（草稿）

在写手窗口输入任务，包含：
- 格式（文章/邮件/章节/脚本）
- 主题
- 受众与目标
- 参考材料
- RULES.md 中的规则

先修结构，别管细节。

### 第二步：召唤裁判审查

打开新窗口，切换模型（如可用），给这个指令：

```
你在审查一份草稿是否存在AI slop和风格问题。不要重写它。这是我们的规则：

[粘贴 RULES.md 全部内容]

这是草稿：[粘贴V0内容]

请标出这份草稿哪些地方违反了规则。对每个问题，用一句很短的话建议怎么修。
```

### 第三步：生成V1（修订版）

回到写手窗口：

```
这是你原来的草稿，以及一份指出问题的审查报告。
[粘贴审查报告]

请重写这份草稿，只修这些问题。保持结构和主要论点不变。
```

### 第四步：人类最后一遍（V2）

这是最关键的步骤。你才是稿件真正的主人。

- **读结构**：现在的段落顺序，是你口头给朋友讲故事的逻辑吗？
- **读准确性**：数字、人名、结论是你确认过的吗？AI经常一本正经地胡说八道。
- **读「质感」**：这篇文章看起来像你写的吗？是否符合你的风格或主张。
- **注入个性**：把你的立场、态度、声音注入文章。

如果发现新的AI口头禅，反手加进你的「词汇黑名单」。

## 版本思维

- V0：AI搭的架子
- V1：AI自己互评修好的干净稿子
- V2：你注入灵魂的最终作品
