# Humanize Write v2

把 AI 草稿变成「读起来自然专业」成品的优化流程。

**核心升级**：v2 在裁判和写手之间加入了「风格校准器（Moderator）」，防止过度润色导致口语化过重，并支持自由迭代循环直到你确认满意。

---

## 快速开始

### 方式一：直接写（新手友好）

在写手窗口输入：

```
场景：[学术论文/商业计划/课程作业/演讲稿/私人邮件]
主题：[你的主题]
受众：[受众描述]
目的：[目标]
口语温度偏好：[1-10，越高越口语化，不填默认3-4]

参考材料：[粘贴笔记/大纲/数据]

规则：见 RULES.md
```

### 方式二：两窗口工作流（推荐）

**写手窗口** — 生成草稿
**Moderator窗口** — 风格校准（防止过度润色）
**裁判窗口** — 规则审查（AI套话检查）

---

## 完整工作流（自由迭代循环）

```
V0 → Moderator审查 → 裁判审查 → V1
  ↑                              ↓
  ← ← ← ← ← ← ← ← ← ← ← ← ← ← ←
           （自由迭代）
                ↓
             你确认
                ↓
               V2
```

### 第一步：写手生成 V0

输入任务，包含：
- 场景/格式（文章/邮件/章节/脚本）
- 主题、受众与目标
- 参考材料
- 场景温度偏好（参考 MODES.md）
- RULES.md 中的规则

先搭结构，别管细节。

### 第二步：风格校准（Moderator）

打开新窗口（新模型），输入：

```
你是风格校准器。检查这篇文字是否存在「过度口语化」和「AI套话残留」问题。
结合场景判断风格是否合适。

场景：[论文/计划/作业/演讲/邮件]
口语温度偏好：[数字]

这是规则：见 MODERATOR.md
这是文字：[粘贴 V0 内容]

输出格式见 MODERATOR.md，不要重写。
```

Moderator 会给出：
- 口语温度值（1-10）
- 过度口语化问题列表
- AI套话残留问题列表
- 风格平衡判断

### 第三步：规则审查（裁判）

继续在 Moderator 窗口或开新窗口，输入：

```
你在审查一份草稿是否存在AI套话和规则违反。不要重写它。

这是规则：见 RULES.md
这是草稿：[粘贴 V0 内容]

请标出违规问题，对每个问题用一句话建议怎么修。
```

### 第四步：生成 V1

回到写手窗口：

```
这是你原来的草稿，以及两份审查报告。

[粘贴 Moderator 报告]
[粘贴 裁判报告]

请重写这份草稿：
1. 只修这些问题
2. 保持结构和主要论点不变
3. 注意 Moderator 指出的「过度口语化」问题 — 不要为了去AI味而变得太随意
4. 注意裁判指出的「AI残留」问题 — 不要过度使用口语化替换

重写后，这篇文字应该听起来「自然专业」，而不是「刻意随意」。
```

### 第五步：自由迭代循环

V1 输出后，会询问你：

> **「还需要调整吗？可以说具体哪里不满意，比如：**
> - **『第三段太口语了，改专业一点』**
> - **『第二段加个例子』**
> - **『结尾太弱，加强说服力』**
> - **或者直接说『可以了』结束」**

**如何反馈**：
- 说具体问题 → AI 修订 → Moderator 审查 → 新V1 → 再询问
- 说「可以了」→ 进入最终确认

这个循环可以跑任意次，直到你满意。

### 第六步：最终确认（V2）

**你才是稿件真正的主人。**

- **结构**：段落顺序是你口头给朋友讲这个故事的逻辑吗？
- **准确**：数字、人名、结论是你确认过的吗？AI 会一本正经地胡说八道。
- **质感**：这篇文章看起来像你写的吗？符合你的立场和风格吗？
- **平衡**：是不是太专业了（听起来像AI）？还是太随意了（像在发微信）？

确认无误后，这就是你的 V2。

## 版本思维

| 版本 | 职责 | 谁做 |
|---|---|---|
| V0 | AI搭架子 | 写手 |
| V1 | AI自修 + Moderator校准 + 裁判审查 | 写手+Moderator+裁判 |
| V2 | 你注入灵魂的最终作品 | 你 |

---

## 三种工作模式

### 快速模式（简单任务）
直接给写手：「场景 + 主题 + 参考材料 → V0 → 你确认 → 直接V2」

### 标准模式（一般任务）
走完 V0 → Moderator → 裁判 → V1 → 迭代 → 你确认 → V2

### 深度模式（重要任务）
V0 → Moderator → 裁判 → V1 → 迭代 → Moderator → V1' → 再迭代 → 你确认 → V2
（每个版本都过 Moderator + 裁判双重审查）

## 场景选择（MODES.md）

如果不确定场景风格：
- 学术论文 → 温度 2-3，正式严谨
- 商业计划 → 温度 3-4，专业有说服力
- 课程作业 → 温度 4-5，学生腔但不过度
- 演讲稿 → 温度 5-6，口语化可朗读
- 私人邮件 → 温度 6-7，轻松自然

详细说明见 MODES.md。
